Motivace: proč přejít na TypeScript

Pro vývojáře zvyklé na JavaScript přináší TypeScript řadu výhod, které pomáhají vytvářet robustnější a udržitelnější aplikace.

// JavaScript - bez typů, potenciální problémy jsou odhaleny až za běhu
function vypoctiCenu(mnozstvi, cenaZaKus) {
  return mnozstvi * cenaZaKus;
}

// Funguje správně
vypoctiCenu(5, 100); // 500

// Neočekávané chování
vypoctiCenu("5", 100); // "5100" - řetězec se zřetězí s číslem místo násobení
vypoctiCenu(undefined, 100); // NaN - bez varování v době vývoje

Co je tsconfig.json?

tsconfig.json je konfigurační soubor pro TypeScript kompilátor. Umožňuje ti nastavit, jak má TypeScript překládat kód, které soubory má brát v potaz a jaké pravidla má uplatňovat. Tento soubor se automaticky vytvoří příkazem:

tsc --init

Typický obsah vypadá následovně:

{
  "compilerOptions": {
    "target": "ES6",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}

Důležité volby v compilerOptions

  • strict: Zapne přísné typování – doporučeno pro maximální bezpečnost.
  • esModuleInterop: Umožňuje kombinovat importy z CommonJS a ES modulů.
  • target: Určuje verzi JavaScriptu, do které se má kód přeložit (např. ES6).
  • module: Nastavuje modulový systém (např. commonjs pro Node.js).

První kroky při migraci

Začni přejmenováním souborů z .js na .ts. Můžeš také povolit podporu JavaScriptových souborů v projektu, což umožní postupnou migraci:

{
  "compilerOptions": {
    "allowJs": true,
    "checkJs": true
  },
  "include": ["src/**/*"]
}

Veškeré typescript soubory je důležité ukládat do složky /src, které se pak zkompilujou na JavaScript ve složce /dist

Postupné přidávání typů

Nemusíš vše přepisovat najednou. Začni přidávat typy do nového nebo důležitého kódu. TypeScript ti pomůže odhalit chyby, i když část kódu zůstane dočasně v JavaScriptu.
